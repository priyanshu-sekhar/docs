## Pulling Data

When data is shared with a client, a custom schema, views, and functions (if applicable) are provisioned. Views are used to provide intraday tick data and functions are used to provide snapped data. These allow you to retrieve data based on your license agreement. In your client schema you will also have a records view which contains the records you are permissioned for.

### 1. Searching for Records

You can search for records using wildcards on the record description. The records view in your schema shows you only the records you are entitled for.

The `market_group_name` (equivalent to Commercial Package Name) returned in the query indicates the name of the object/commercial package the data is available in.

<CodeGroup>

```java Swaption ATM records

SELECT
  *
FROM
  COMMERCIAL_PACKAGES.SHARE_CLIENT_X.RECORDS
WHERE
  description ILIKE '%swaption%'
  AND description ILIKE '%ATM%';
```

```java Payer/Receiver ATM +100 records
SELECT
  *
FROM
  info.records
WHERE
  description ILIKE '%swaption%'
  AND description ILIKE '%+ 100%';
```

</CodeGroup>

### 2. Searching For Fields on a Record

Each record has a set of fields available on it dependent upon it's stub (the first five characters of the record name). The fields for each record can be retrieved using the field group ID or the stub (field group name).

<Note>
  **Note:** Here 'Field Group' is represented by the first 5 characters of the record name. For more information, see [Field Groups Overview](/tabs/indicative-data/our-data/field-groups-overview)
</Note>

```java
SELECT 
  * 
FROM 
  COMMERCIAL_PACKAGES.INFO.FIELD_GROUP
WHERE 
  NAME = 'IVSWR';
```

This allows us to link related records and provide the complete history.

## What's in the Info Schema?

### Record Metadata Definitions

Views describing the meaning of each acronym in record names.

- LOCATIONS
- CLEARING\_TYPES
- PRICE\_TYPES
- PRICING\_FREQUENCIES
- COMPANIES

For more information, see [Record Naming Convention](/tabs/indicative-data/our-data/record-naming-convention)

### Field Groups

The first 5 characters of the record name indicates the field group it abides to. This can also be joined via the **FIELD\_GROUP\_ID** column.

For more information, see [Field Groups Overview](/tabs/indicative-data/our-data/field-groups-overview)
