## Standard Message Fields

Present on every trade message regardless of asset class.

| Field | Type | Description | Example |
| --- | --- | --- | --- |
| `EVENT_TIMESTAMP` | datetime | Processing timestamp of the event (UTC) | `2026-06-09T11:36:08.628` |
| `GUID` | string | Globally unique event identifier | `dbefd8fa-3760-4c59-8881-e418eb71e5e9` |

## Standard Trade Fields

Present on every trade message regardless of asset class.

| Field | Type | Description | Example |
| --- | --- | --- | --- |
| `DEAL_ID` | string | Unique deal identifier (stable across lifecycle) | `11112958` |
| `TRADE_ID` | string | Internal TP ICAP trade ID — multiple trades may roll up into a deal | `3521932` |
| `EXECUTION_CHAIN_ID` | string | Links related events in an execution chain | `23416338` |
| `DEAL_STATUS` | string | Lifecycle status: `NEW`, `AMEND`, `CANCEL` | `NEW` |
| `DEAL_EXECUTION_DATETIME` | datetime | Trade execution timestamp | `2026-06-09T10:43:47.640` |
| `ASSET_CLASS_ISDA` | string | ISDA Taxonomy 2.0 — Asset Class | `Commodity` |
| `BASE_PRODUCT_ISDA` | string | ISDA Taxonomy 2.0 — Base Product | `Energy` |
| `SUB_PRODUCT_ISDA` | string | ISDA Taxonomy 2.0 — Sub-Product | `Oil` |
| `TRANSACTION_TYPE_ISDA` | string | ISDA Taxonomy 2.0 — Transaction Type | `Future` |
| `BROKING_METHOD` | string | Method of brokerage: `VOICE`, `ELECTRONIC`, `HYBRID` | `Voice` |
| `VENUE` | string | Trading venue MIC code | `IFEU` |
